本手册是 hait6.0 的一部分。

hait6.0主要是采用css3/javascript/jquery1.10/html5语言写成，本手册针对的是web界面开发及相关人员。对于相关的基础知识，本手册不做专门的介绍。
本手册针对的浏览器运行环境，推荐使用谷歌14+，火狐10+，IE10+。

CSS编码规范

选择器存在多个单词时使用减号"-"来进行连接,这是HTML5标准规范

.hait-select {
    width: 200px;
    height: 28px;
}

.hait-select .select-show-text {
    width: 160px;
    height: 26px;
}

HTML编码规范

1.属性存在多个单词时使用减号"-"来进行连接,这是HTML5标准规范
2.编号(id)和名称(name)由于要用于JS操作,所以采用骆驼命名法,这是编程语言标准规范
3.功能号(funcId)参见后面的hait框架编码规范
4.接口字段参见后面的hait框架编码规范

JS编码规范

1.属性方法均采用骆驼命名法,这是编程语言标准规范
2.私有属性,前端增加下划线
3.事件属性,以on前缀 

hait.form.field.SelectField = function () {
    /**
     * mode为func时的相关参数
     */
    this.funcId = null;
    this.params = null;

    /**
     * 放置事件对象
     */
    this.onchange = null; // 选择变更事件
    this.onrefresh = null; // 完成刷新后执行的事件

    /**
     * 私有属性
     */
    this._isInitComplete = false; // 是否已经初始化完毕
    this._delayValue = null;// 延迟值（用于初始化未完成时的赋值操作）
};

Hait框架编码规范

1.功能号(funcId),以hex为前缀,中间单词为所在文件名称(可能多个),后面才是详细功能号名称,使用骆驼命名法
    示例:hex_cust_account_queryCustAccountById
    这个功能号位于cust文件夹下的account.xml文件中,名称为queryCustAccountById
2.数据库表名和字段名均采用小写单词,多个单词采用下划线隔开
